# CH 2 계산기 과제

Java의 제네릭과 스트림을 활용하여 정수와 실수를 모두 처리할 수 있는 유연한 계산기 프로젝트입니다.

## 핵심 기능

1. 제네릭 기반의 숫자 연산
- `T extends Number`를 활용하여 Integer, Double 등 다양한 숫자 타입을 수용합니다.
- 내부적으로 doubleValue()를 사용하여 정밀한 연산을 수행합니다.

2. Enum을 활용한 연산자 관리
- `OperatorType` 열거 타입을 통해 연산 기호(`+`, `-`, `*`, `/`)를 객체화하여 관리합니다.
- 입력값 검증 로직을 포함하여 잘못된 기호 입력 시 안전하게 예외를 발생시킵니다.

3. 지능형 결과 필터링 (Stream API)
- 저장된 연산 결과 중 사용자가 입력한 기준값(`input`)보다 큰 결과만 선별하여 출력합니다.
- Java Stream과 Lambda를 사용하여 선언적인 데이터 처리를 구현했습니다.

4. 사용자 친화적 출력
- 결과값이 정수로 떨어지는 경우(예: `5.0`) 소수점을 제거하여 `5`로 출력하는 포맷팅 로직이 포함되어 있습니다.

## 셀프 피드백

### 잘한점
- 목적에 맞는 브랜치를 생성하고 기능 단위에 맞게 커밋을 작성했습니다.
- enum을 단순 열거타입이 아닌 익명 함수로도 활용 할 수 있다는것을 배웠습니다.

### 아쉬운점
- 요구 사항에 맞게 코드를 수정할때 다 같은 커밋 메시지를 작성하다보니 커밋을 살펴보지 않으면 무엇이 어떻게 변경되었는지 알기 어렵게 되었습니다. 다음에는 좀 더 구분이 되도록 작성해봐야겠습니다.
- 요구 사항을 제대로 분석하지 않고 커밋을 신중하게 하지 않아 불필요한 커밋이 많이 생겼습니다. 다음에는 요구 사항을 더 자세히 봐야겠습니다.
- 자잘한 커밋이 너무 많아졌을때는 git squash로 커밋을 합쳐보는것도 좋을거같다.
